## 🎯 What It Does

1. **Reads** UDE/employee data from CSV files
2. **Maps** User_ID → SAPID via Resources.csv
3. **Transforms** data into API-ready format
4. **Authenticates** with JWT tokens
5. **Submits** batch to TeamSight API
6. **Saves** generated JSON for verification

## 🔑 CLI Arguments

```
--resources           Path to Resources.csv (required)
--ude-data            Path to UDE sheet CSV (required)
--extension-data      Path to version availability CSV (optional)
--api-url             TeamSight API base URL (required)
--api-user            API username (default: api_user)
--api-password        API password (required)
--output-dir          Where to save JSON files (default: ./ude_payloads)
--dry-run             Generate payloads without submitting
```

## ⚠️ Known Limitations

- Unmapped User IDs are **skipped** (90 records in test data)
  - Solution: Update UDE system to use UDEID values from Resources.csv
- Device IDs are **dummy values** (LAPTOP-HCL-{initials}{hash})
  - Solution: Extend program to accept real hardware IDs
- Status always set to **"installed"**
  - Solution: Add `--status` parameter or extend CSV schema
