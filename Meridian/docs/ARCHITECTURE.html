TeamSight — System Architecture

KPI tracking and engineering metrics platform — 27+ KPIs across individual, scrum, and team levels, computed from GitHub, JIRA, and Copilot data, served via a React dashboard.

◈ Overview

Data Ingestion Scheduled
Incremental fetchers pull GitHub commits, JIRA issues, Copilot metrics, and security scan reports on configurable cron schedules.
KPI Computation 27 KPIs
41 Python scripts compute individual, scrum, and team metrics, writing standardised CSVs keyed by SAPID.
API Layer FastAPI
REST API with JWT auth and RBAC aggregates KPI data per team/scrum/employee and serves scoring, reports, and admin endpoints.
Dashboard React 18
React SPA with 15 pages including KPI matrix, drill-down dashboards, admin scheduler UI, and role management.

4 KPI Computation Engine src/

Python 3.9 pandas csv datetime
KppEvaluator.py Orchestrator
Registers all 27 KPIs. Dispatches --run-all or --kpi k268. Handles equivalent KPI mappings: k8→k4, k20→k16, k22→k18.
kpp_k*.py 41 scripts
One script per KPI. Reads raw output CSVs + Resources.csv. Writes output/k*-data.csv in the standard schema.
kpi_helpers.py Shared
Fiscal week/month/quarter/year string generators. Shared bug-counting utilities used across KPI scripts.

Standard KPI Output Schema

CurrentDate, Week, Month, Quarter, Year, SAPID, Name, Weekly, Monthly, Quarterly, Annual
Team-level ratio aggregation (k268, k222): Computed as sum(numerator members) / sum(denominator members) over the applicable-role member set — not as average of individual ratios. This avoids 9999 sentinel values (emitted when Copilot LOC = 0) inflating the team average.

APScheduler ≥ 3.10 ThreadPoolExecutor JSON state persistence

Deployment Steps

1
Bump version in setup.py, then rm -rf dist && python -m build --wheel
2
Package bundle (code-only, never overwrites remote data): ./scripts/package_complete_bundle.sh --no-prompt
3
SCP bundle to remote: scp dist/teamsight-complete-*.tar.gz root@10.222.2.217:/tmp/
4
Deploy on remote: extract bundle → run deploy_complete_bundle.sh --install-wheel --start-services
5
Rebuild frontend on remote: cd dashboard/frontend && npm run build
6
Restart backend: ./dashboard/manage.sh restart backend
7
Verify: ./manage.sh status · open http://10.222.2.217:8000/
Safe deployment guarantee: The standard bundle excludes config/, data/, and output/. Live CSV data, scheduler state, and user files on the remote server are never overwritten by a routine code deployment.
