KPI Engine Python · src/ Slide 4 of 9
Component File / Module Role Details
KPI Orchestrator KppEvaluator.py Central dispatcher Registers 27 KPIs, resolves equivalents (k8→k4, k20→k16, k22→k18), loads inputs, dispatches per-KPI runs. CLI entry: teamsight-kpi
KPI Scripts kpp_k*.py (50+ files) Individual KPI logic One file per KPI metric. Each reads Resources.csv + Roles.csv + raw source CSVs and emits a standardised 11-column output CSV to output/k*-data.csv
KPI Helpers kpi_helpers.py Shared utilities Fiscal calendar logic, ISO week / month / quarter label generation, SAPID string normalisation
JIRA Fetcher jira_fetch.py Data ingestion JIRA REST API v2/v3 — incremental fetch with checkpoint. Writes JIRAIssues.csv. CLI: teamsight-jira-fetch
GitHub Fetcher github_fetch.py Data ingestion GitHub REST API — commits, PRs, reviews. Maps GitHub usernames to SAPID via Resources.csv. CLI: teamsight-github-fetch
Copilot Metrics Fetcher copilot_metrics_fetch.py Data ingestion GitHub Copilot Metrics API (org/enterprise level). Fetches active users, suggested lines, acceptance rates
Security Scan Fetcher security_scan_fetch.py Data ingestion Config-driven integration with SAST/DAST tools. Project→team mappings in security_scan_config.json
Bug Cycle Time kpp_bug_cycle_time.py Specialised report Computes resolution cycle time for JIRA bug tickets. Used by the Bug Cycle Time report page
Replan Tracker kpp_replan_tracker.py Specialised report Detects sprint replanning events from JIRA sprint history changelog
Assignee Delay kpp_assignee_attributable_delay.py Specialised report Computes delay windows attributable to assignee inactivity per JIRA issue
Epic Tree Cache Job jira_epic_tree_cache_job.py Background job Pre-computes and caches the JIRA epic → story → task hierarchy for the tree report
Daily Backup Job daily_backup_matrix_job.py Maintenance job Automated daily backup of KPI matrix data to backup/ directory

Security Auth · RBAC · Transport Slide 7 of 9
Mechanism Technology Where Applied Details
JWT Authentication python-jose HS256 All API endpoints (except /auth/login, /health) Bearer tokens signed with HS256. 24-hour expiry. Validated via get_current_user FastAPI dependency
Password Hashing passlib / bcrypt User credentials in users.json Passwords never stored in plain text. bcrypt with auto-generated salts; CryptContext for hash verification
Role-Based Access Control FastAPI dependencies Protected API routes Four roles: Admin, API User, Viewer, Manager. Enforced via require_role(), require_permission(), require_admin() dependencies
CORS Middleware FastAPI CORSMiddleware All cross-origin requests Configurable allowed origins via cors_origins setting. Development: wildcard; production: restrict to deployment domain
Secret Management python-dotenv + pydantic-settings JWT secret, API tokens All sensitive values loaded from dashboard/backend/.env. Never hardcoded in source
Input Validation Pydantic v2 All API request bodies & query params Full runtime type validation before reaching business logic. Prevents injection-style attacks at system boundary
Global Error Handler FastAPI exception_handler Uncaught exceptions Returns sanitised JSON error (no stack traces in production). Logs full context server-side
